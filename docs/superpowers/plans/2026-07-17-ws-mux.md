# WebSocket Multiplexing (`cairn-mux-v0`) Implementation Plan

> **For agentic workers:** Hybrid execution — the main agent implements tasks marked
> **Owner: main**; tasks marked **Owner: subagent** are dispatched to fresh subagents and
> reviewed by the main agent. Tasks use checkbox syntax for tracking. Spec:
> `docs/superpowers/specs/2026-07-17-ws-mux-design.md` — read it before any task; it is
> the authority on wire-protocol semantics.

**Goal:** One persistent WebSocket carries all unary wRPC traffic from the web UI via a
cairn-owned channel-mux subprotocol, negotiated with `Sec-WebSocket-Protocol`, with the
one-shot protocol retained for dedicated streams.

**Architecture:** A 5-byte-header framing layer (`[channel_id: u32 BE][flags: u8]`) turns one
WebSocket into N logical byte streams, each carrying exactly one wRPC invocation. Rust side: a
`MuxAcceptor` implementing `wrpc_transport::frame::Accept` feeds the same accept-loop pattern as
`serve/wrpc.rs`. TS side: a `wsMuxDialer` returns one `Transport` per channel over a shared
cached socket, mirroring `wtDialer`'s shape. No changes to `wrpc-transport` 0.29, the vendored
JS package, WIT, handlers, or the registry.

**Tech Stack:** Rust (tokio, tokio-websockets via `wrpc_websockets` re-export, wrpc-transport
0.29, axum/hyper upgrade), TypeScript (`@bytecodealliance/wrpc` `Transport`/`Chan`, vitest),
existing daemon integration harness (`crates/cairn-daemon/tests/common`) and web node harness
(`cairn-web/tests/integration`).

## Global Constraints

- Wire constants (spec is authoritative): header = `[channel_id: u32 big-endian][flags: u8]`;
  flags bit 0 = `FIN`, bit 1 = `RST`, bits 2–7 reserved (non-zero ⇒ protocol error); channel
  id 0 reserved (⇒ protocol error); ids strictly increasing, client-opened only; max payload
  1 MiB; max 256 concurrent channels (excess ⇒ daemon RSTs the new channel); text frame in
  muxed mode ⇒ protocol error; stale frames (non-live id ≤ highest seen) silently ignored.
- Subprotocol names: `cairn-mux-v0`, `cairn-oneshot-v0`. No offer ⇒ one-shot, no echo.
- Daemon WS ping interval on muxed connections: 30 s.
- No `unwrap`/`expect`/`panic!` in production Rust (tests exempt). Errors propagate.
- Tests assert behavior through real interfaces, never code shape.
- Verify each task with: `cargo nextest run -p cairn-daemon`, `cargo clippy --all-targets --
  -D warnings`, `cargo fmt --check` (Rust); `npm test` in `cairn-web/` (TS). Pre-commit hooks
  (biome, clippy, fmt, svelte-build) must pass on commit.
- Commit per task, message explains *why*.

### Task 1: Rust mux core (`ws_mux` module)

**Owner: main** (protocol state machine, async subtleties: cancel safety, backpressure,
single-writer discipline)

**Files:**
- Create: `crates/cairn-daemon/src/serve/transport/ws_mux.rs`
- Modify: `crates/cairn-daemon/src/serve/transport/mod.rs` (register module)

**Interfaces:**
- Consumes: `wrpc_transport::frame::Accept` (the trait `OneShot` implements today —
  see `websocket.rs:180` for the shape), `crate::serve::ConnCtx`.
- Produces (used by Task 2 and by Task 2's integration-test client):
  - `pub(crate) async fn serve_mux<S>(ctx: ConnCtx, ws: S, daemon: crate::daemon::Daemon, shutdown: CancellationToken) -> anyhow::Result<()>`
    where `S` is the `tokio_websockets` WebSocket stream (`Sink<Message> + Stream<Item =
    Result<Message, _>>`) — generic over the stream so tests can drive it over
    `tokio::io::duplex`-backed WebSocket pairs.
  - `pub(crate) mod frame` helpers: `encode_frame(channel_id: u32, flags: u8, payload: &[u8]) -> Bytes`
    and `decode_frame(msg: &Bytes) -> Result<(u32, u8, Bytes), ProtocolViolation>` plus
    `pub(crate) const FIN: u8`, `RST: u8`, `MAX_PAYLOAD: usize`, `MAX_CHANNELS: usize` —
    shared with the test client so the codec isn't duplicated.

**Outcome:** The muxed serve path from the spec's "Daemon design" section: one reader task
(frame parse, channel routing, open-on-new-id, tolerance rule, limits, whole-socket pause on a
full channel buffer), one writer task (sole sink consumer, flush-after-write, 30 s pings), a
per-channel logical `AsyncRead`/`AsyncWrite` pair (`poll_shutdown` ⇒ FIN; dropping an un-FIN'd
writer ⇒ RST), a `MuxAcceptor` implementing `Accept` fed by the reader, and an accept loop
mirroring `serve/wrpc.rs::run_wrpc_server` (each invocation spawned, cancelled on shutdown).
Handlers are registered via `cairn_protocol::serve` exactly as in `serve_one`.

**Acceptance criteria** (behavioral tests in the same file or `ws_mux` test module, driven over
an in-memory WebSocket pair speaking raw frames — no real daemon needed; a stub `Daemon` or the
real one with an empty registry both work):
- [ ] Two interleaved invocations on one connection both complete with correct results.
- [ ] Client FIN half-close still receives full results (FIN piggybacked on a data frame and
      standalone empty-payload FIN both work).
- [ ] RST mid-call kills only that channel's invocation; a concurrent call is undisturbed.
- [ ] Frames for a closed/RST channel (id ≤ highest seen, not live) are silently ignored.
- [ ] Each protocol violation — reserved flag bits, text frame, payload > 1 MiB, channel id 0 —
      closes the connection (all live invocations end).
- [ ] Channel 257 (concurrent) is RST'd; the socket and the other 256 stay alive.
- [ ] Daemon sends a WS ping within the ping interval (tolerances at the implementer's
      discretion — assert traffic, not timing precision).
- [ ] Shutdown cancellation ends the serve loop without hanging.
- [ ] `cargo nextest run -p cairn-daemon`, clippy, fmt all clean. Commit.

### Task 3: TS `wsMuxDialer`

**Owner: main** (async error paths, socket lifecycle, channel table correctness)

**Files:**
- Create: `cairn-web/src/lib/protocol/wsmux.ts`
- Create: `cairn-web/src/lib/protocol/wsmux.test.ts`
- Modify: `cairn-web/src/lib/protocol/index.ts` (export)

**Interfaces:**
- Consumes: `Transport`/`Dialer` from `./transport`, `Chan` from `@bytecodealliance/wrpc`,
  the `bufferedAmount` backpressure approach from `ws.ts` (extract/reuse
  `writeWithBackpressure` rather than duplicating it).
- Produces: `export function wsMuxDialer(url: string, opts?: { onDown?: (err: Error) => void }): Dialer`
  (consumed by Task 5).

**Outcome:** The spec's "Web client design" dialer: one lazily-opened cached
`WebSocket(url, ['cairn-mux-v0'])` with `binaryType = 'arraybuffer'`; verifies `ws.protocol ===
'cairn-mux-v0'` on open and fails otherwise; per dial allocates the next channel id (from 1) and
returns a `Transport` (read ⇐ per-channel `Chan` fed by the demux; write ⇒ header-prefixed
frames with backpressure; `closeWrite` ⇒ empty FIN; `close` ⇒ RST-if-incomplete + free entry).
`onclose`/`onerror` fails all live channels' Chans, clears the table, forgets the socket
(next dial redials), and fires `onDown` once per socket death.

**Acceptance criteria** (vitest, mock WebSocket class injected the way `protocol.test.ts`
arranges its fakes — check that file's pattern first):
- [ ] Frames written carry correct 5-byte headers; two concurrent dials use distinct increasing
      ids on the same (single) socket.
- [ ] Demux routes payloads to the right channel's `read()`; peer FIN ⇒ EOF (`undefined`);
      peer RST ⇒ read rejects.
- [ ] `closeWrite()` sends empty-payload FIN; `close()` on an incomplete channel sends RST;
      `close()` after clean completion sends nothing.
- [ ] Stale inbound frames (unknown non-live id) are ignored without error.
- [ ] Socket drop rejects all live channels' pending reads, fires `onDown` once; the next dial
      opens a fresh socket.
- [ ] Negotiation failure (mock reports wrong/empty `ws.protocol`) rejects the dial.
- [ ] `npm test` (and repo lint hooks) clean. Commit.

### Task 5: Connection wiring + reconnect polish

**Owner: subagent** (small, but has one precise behavioral addition — `kick()`)

**Files:**
- Modify: `cairn-web/src/lib/stores/connection.svelte.ts`
- Modify: `cairn-web/src/lib/stores/reconnect.ts`
- Modify: `cairn-web/src/lib/stores/reconnect.test.ts` (or create if absent — check first)

**Interfaces:**
- Consumes: `wsMuxDialer(url, { onDown })` (Task 3), two-dialer `DaemonClient` (Task 4).
- Produces: `ReconnectController.kick(): void` — public method: if stopped or a probe is
  already in flight, no-op; otherwise cancel any pending timer and run the probe immediately.
  (Needs an in-flight guard flag; `#runNow` currently has no reentrancy protection.)

**Outcome:** For WS endpoints, `connectWith` builds `control = wsMuxDialer(url, { onDown: () =>
controller.kick() })` and `streams = wsDialer(url)`; WebTransport endpoints pass `wtDialer` for
both roles. Steady probe interval goes 15 s → 30 s (`steadyIntervalMs` at the construction
site; update the doc comment default note if the default itself changes). Result: socket death
flips status to `reconnecting` on the next tick instead of up to a probe-interval later.

**Acceptance criteria:**
- [ ] `kick()` behavior pinned by unit tests with injected scheduler: immediate re-probe when
      idle-connected; no-op while stopped; no concurrent double-probe when called mid-probe.
- [ ] Connection store test (or existing suite) still passes; WS path constructs both dialers.
- [ ] `npm test` clean. Commit.
